<UpdateSection type="upgrades">
    <IconList>
        <IconItem icon="Wrench" color="#EEB03D">
            Added sorting by country/downloads/last updated (default is now last updated)
        </IconItem>
        <IconItem icon="Wrench" color="#EEB03D">
            All assets now show download counts on the Browse page, with maps also showing flag icons corresponding to the country of the map
        </IconItem>
        <IconItem icon="Wrench" color="#EEB03D">
            Search now only show tags when an asset with that tag exists
        </IconItem>
        <IconItem icon="Wrench" color="#EEB03D">
            Separated toasts for download progress and download success and added an indicator for how many downloads are in queue
        </IconItem>
        <IconItem icon="Wrench" color="#EEB03D">
            Added an updated multistep loading screen on startup
        </IconItem>
        <IconItem icon="Wrench" color="#EEB03D">
            Railyard now launches in fullscreen
        </IconItem>
        <IconItem icon="Wrench" color="#EEB03D">
            Linux builds are now through Flatpak. This means you will need to have Flatpak installed in order for the app to function. In addition, this means the autoupdater will be broken for one patch on Linux (you will need to manually install the flatpak included in the v0.1.4 release yourself)

            :::note

            You may not be able to find normal file paths when using the flatpak on linux. This is normal. Your host file system will be mounted on /run/host. You **will** have to update your paths when switching to the flatpak build.

            :::
        </IconItem>
    </IconList>
</UpdateSection>
